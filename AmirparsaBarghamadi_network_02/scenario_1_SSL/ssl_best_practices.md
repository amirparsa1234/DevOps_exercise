# بهترین روش‌های SSL و TLS

## SSL و TLS
SSL قدیمی است و دیگر توصیه نمی‌شود.  
TLS استاندارد امروزی برای ارتباط امن است.

## گواهی Self-signed و CA-signed
Self-signed یعنی گواهی را خودِ سرور امضا کرده است. برای تست و محیط آزمایشگاهی خوب است، ولی معمولاً مرورگرها به آن اعتماد نمی‌کنند.  

CA-signed یعنی گواهی توسط یک مرجع معتبر صادر شده است. اگر دامنه درست باشد و زنجیره گواهی کامل باشد، مرورگرها بدون هشدار آن را قبول می‌کنند.

## بهترین روش‌ها
1) فقط از نسخه‌های امن TLS استفاده کن.  
2) الگوریتم‌های رمزنگاری قوی را فعال نگه دار.  
3) اعتبار گواهی و تطابق نام دامنه را همیشه بررسی کن.  
4) تاریخ انقضا را کنترل کن و قبل از تمام شدن تمدید کن.  
5) برای سایت‌ها اجبار اتصال امن را فعال کن.

## HTTP و HTTPS
HTTP اطلاعات را ساده و قابل مشاهده ارسال می‌کند و امکان تغییر در مسیر وجود دارد.  
HTTPS ارتباط را رمزنگاری می‌کند و باعث می‌شود داده‌ها امن‌تر منتقل شوند و هویت سرور بهتر تأیید شود.

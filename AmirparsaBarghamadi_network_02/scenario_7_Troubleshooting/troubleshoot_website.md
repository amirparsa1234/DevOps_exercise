# راهنمای عیب‌یابی و توضیحات شبکه

## بهترین روش‌های SSL و TLS

### SSL و TLS
SSL قدیمی است و دیگر توصیه نمی‌شود.  
TLS استاندارد امروزی برای ارتباط امن است.

### گواهی Self-signed و CA-signed
Self-signed یعنی گواهی را خودِ سرور امضا کرده است. برای تست و محیط آزمایشگاهی خوب است، ولی معمولاً مرورگرها به آن اعتماد نمی‌کنند.  
CA-signed یعنی گواهی توسط یک مرجع معتبر صادر شده است. اگر دامنه درست باشد و زنجیره گواهی کامل باشد، مرورگرها بدون هشدار آن را قبول می‌کنند.

### بهترین روش‌ها
1) فقط از نسخه‌های امن TLS استفاده کن.  
2) الگوریتم‌های رمزنگاری قوی را فعال نگه دار.  
3) اعتبار گواهی و تطابق نام دامنه را همیشه بررسی کن.  
4) تاریخ انقضا را کنترل کن و قبل از تمام شدن تمدید کن.  
5) برای سایت‌ها اجبار اتصال امن را فعال کن.

### HTTP و HTTPS
HTTP اطلاعات را ساده و قابل مشاهده ارسال می‌کند و امکان تغییر در مسیر وجود دارد.  
HTTPS ارتباط را رمزنگاری می‌کند و باعث می‌شود داده‌ها امن‌تر منتقل شوند و هویت سرور بهتر تأیید شود.

## مقایسه سریع انتقال فایل

### FTP
- پورت: ۲۱  
- به صورت پیش‌فرض رمزنگاری ندارد  
- ساده و قدیمی است و هنوز روی بعضی سیستم‌های قدیمی دیده می‌شود  
- برای اینترنت عمومی مناسب نیست چون نام کاربری، رمز عبور و داده‌ها می‌توانند قابل شنود باشند  

### SFTP
- پورت: ۲۲  
- رمزنگاری شده است  
- انتقال فایل روی SSH انجام می‌شود  
- انتخاب پیش‌فرض خوبی برای انتقال فایل امن است  

### SCP
- پورت: ۲۲  
- رمزنگاری شده است  
- ابزار ساده برای کپی فایل شبیه دستور cp ولی روی SSH  
- برای کپی سریع و یک‌باره خوب است و امکاناتش از SFTP کمتر است  

### چه زمانی از کدام استفاده کنیم
- برای بیشتر انتقال‌های امن از SFTP استفاده کن  
- برای کپی سریع فایل‌ها از SCP استفاده کن  
- FTP را روی اینترنت استفاده نکن مگر داخل یک شبکه خیلی کنترل‌شده

## انواع رکوردهای DNS

### A
- رکورد آدرس IPv4  
- مثال: example.com به 1.2.3.4 اشاره می‌کند  

### AAAA
- رکورد آدرس IPv6  

### CNAME
- نام مستعار برای یک نام دیگر  
- مثال: www.example.com به example.com اشاره می‌کند  

### MX
- رکورد سرور ایمیل برای یک دامنه  

### TXT
- رکورد متنی  
- معمولاً برای SPF و DKIM و تأیید مالکیت استفاده می‌شود  

### NS
- رکورد نام‌سرورها برای دامنه  

### SOA
- اطلاعات اصلی زون  
- شامل شماره سریال و زمان‌های refresh و retry و expire
